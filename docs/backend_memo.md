The field study revealed significant gaps in how farmers record and manage operational data, particularly in feed tracking and production monitoring. Based on these insights, several backend adjustments were implemented to better align the system with real-world workflows.

Firstly, simplified API endpoints were designed to support quick and structured data entry, ensuring that farmers can log feed usage and monitoring data with minimal effort. This directly addresses the issue of inconsistent and incomplete data recording observed during the field study.

Secondly, an analytics endpoint was introduced to automatically aggregate and summarize data, providing users with insights such as total feed usage and average larvae growth. This eliminates the need for manual calculations and supports data-driven decision-making.

To enhance trust and reliability, audit logging was implemented to track all user actions. This ensures transparency and accountability within the system, which is essential for building user confidence.

Additionally, role-based access control (RBAC) was added to secure sensitive data and restrict access based on user roles. This improves system security and aligns with best practices in data privacy.

Overall, these backend improvements create a more user-friendly, reliable, and trustworthy system that reflects the actual needs and constraints of farmers, ultimately increasing adoption and impact.
